# Delphi-OBD

> **A complete rewrite — Delphi-OBD v2 — has shipped to public beta on
> the [`v2`](https://github.com/erdesigns-eu/Delphi-OBD/tree/v2)
> branch.**
>
> The v2 line is the future of this project. New features, bug fixes,
> sample improvements, and CI happen on `v2`.
>
> See [v2.0.0-beta.1 release notes](https://github.com/erdesigns-eu/Delphi-OBD/releases/tag/v2.0.0-beta.1)
> and the [v2 README](https://github.com/erdesigns-eu/Delphi-OBD/blob/v2/README.md).

---

The original v1 release of Delphi-OBD is preserved on this `main`
branch as reference only. Code is unchanged from the last v1 tag.

If you have an existing v1 integration:

- **Continue using v1.** The v1 source on this branch will not be
  removed; pin your project to a specific commit.
- **Migrate to v2.** Read
  [`docs/migration-from-v1.md`](https://github.com/erdesigns-eu/Delphi-OBD/blob/v2/docs/migration-from-v1.md)
  on the `v2` branch for the class → component cookbook. The
  catalog file format carries over; AT / ST commands carry over;
  the host API is component-first instead of class-first.

## Issue tracking

Open new issues against the
[v2 branch](https://github.com/erdesigns-eu/Delphi-OBD/issues).

The v1 line is in maintenance mode — security fixes only, on a
case-by-case basis.
