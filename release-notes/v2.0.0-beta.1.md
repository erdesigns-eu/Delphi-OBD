# Delphi-OBD v2.0.0-beta.1

**First public beta of the v2 rewrite.**

This is the candidate for v2.0. Component-first RAD package covering
diagnostics, coding, calibration and ECU flashing across modern
automotive protocols. Use this beta to surface bugs against real
adapters, real ECUs, and your own integrations before the stable
`v2.0.0` ships.

## What's in the box

- **32 design-time components** across five palette tabs
  (`OBD`, `OBD Services`, `OBD Coding`, `OBD Calibration`,
  `OBD Flashing`).
- **26 IDE starter templates** — File → New → Other →
  **Delphi-OBD**. Multi-step wizard with option groups
  (multi-OEM component protection, multi-tab full diagnostics,
  XCP transport choice, DoIP TLS / audit extras, …).
- **Full protocol surface**: UDS, KWP2000, J1939, ISO 13400 DoIP
  (TCP / TLS), AUTOSAR SecOC, ISO 9141-2, ISO 14230, ISO 15765
  (CAN-TP) 11/29-bit, J1850 PWM/VPW, LIN, FlexRay, MOST.
- **Production-grade flash pipeline** with voltage gate, image
  applicability, signature verification (BCrypt / OpenSSL /
  PKCS#11 HSM / post-quantum via liboqs), atomic checkpoint,
  HMAC-chained audit log.
- **Calibration**: ASAM MCD-1 XCP master, ASAP1a CCP master,
  ASAM MCD-2 MC A2L parser.
- **Speciality buses**: ISO 11783 IsoBus (VT / TC / FS / GNSS),
  EU digital tachograph (PC/SC).
- **Recorder / replayer** + `TOBDProtocolMock` for replay-driven
  testing, plus `TOBDLogRedactor` for sharing captures safely.
- **Catalogs** as JSON files (PIDs, DTCs, NRCs, DIDs, J1939
  PGNs / SPNs / FMIs, OEM coding option lists, OEM flash
  handshake overrides, image applicability descriptors).
  Adding entries is a community PR with no recompile.
- **Component editors** with live-test verbs in the IDE
  (Test connection, Detect adapter, Send ATI / AT@1, Validate
  flash configuration), file pickers, COM-port enumerator,
  init-script editor with AT / ST command palette.
- **Documentation suite**: architecture, components reference,
  catalog schemas, migration cookbook, coding cookbook,
  flashing safety, honest phase-by-phase reviews.
- **Sample suite**: 10 standalone projects (smoke tests,
  dry-runs, real-hardware demos).

## ⚠️ Hardware-safety notice

This package can write to and reflash automotive ECUs. **Misuse
can permanently brick an ECU.** Every destructive component
defaults `AutoExecute = False` and refuses to run until the host
either wires `OnConfirmExecute` or explicitly opts in.

**Before you run a flash sample on real hardware, read
[`docs/flashing-safety.md`](https://github.com/erdesigns-eu/Delphi-OBD/blob/v2/docs/flashing-safety.md).**
A stable external charger is required.

## Supported Delphi versions

10.3 Rio through 12 Athens. Win32 / Win64 (the runtime package is
cross-platform clean for everything except the Windows-only
serial / FTDI / Tools-API surface).

## Installation

1. `git clone -b v2.0.0-beta.1 https://github.com/erdesigns-eu/Delphi-OBD`
2. Open `packages/DelphiOBD_RT.dpk` in RAD Studio. Build.
3. Open `packages/DelphiOBD_DT.dpk`. Build, then Install.
4. The **OBD**, **OBD Services**, **OBD Coding**, **OBD
   Calibration**, and **OBD Flashing** categories appear on the
   component palette.
5. File → New → Other → **Delphi-OBD** is now populated with 26
   starter templates.

## Beta scope

The beta window is at least four weeks (extended for the
realities of flashing testing, which requires real ECUs). Bug
reports + feedback during the beta:
[GitHub Issues](https://github.com/erdesigns-eu/Delphi-OBD/issues).

The API surface may shift between `v2.0.0-beta.1` and `v2.0.0`.
Pin to a specific tag in production work. Breaking changes after
v2.0.0 will only land at major version boundaries (semver).

## Known beta-window items

- **No GetIt submission yet.** The GetIt package metadata is
  prepared (`packages/getit-package.json`) but the package will
  be submitted only after the stable `v2.0.0` ships.
- **CI matrix not yet covering RAD Studio builds.** The Windows
  build matrix is configured but gated until a self-hosted
  runner is wired up. Static checks (file headers, VCL/FMX
  guard, JSON catalog lint) run on every PR.
- **Per-component reference pages.** `docs/components.md` is a
  single-page directory; per-component pages
  (`docs/components/<X>.md`) are deferred to v2.1.

See the full [`CHANGELOG.md`](https://github.com/erdesigns-eu/Delphi-OBD/blob/v2/CHANGELOG.md)
for the complete change list and
[`docs/phase-reviews.md`](https://github.com/erdesigns-eu/Delphi-OBD/blob/v2/docs/phase-reviews.md)
for the honest phase-by-phase reviews of every implementation
chunk.
